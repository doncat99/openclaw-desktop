# OntoSynth Course Studio: Artifact and Review Contract (v1)

## Scope

This contract defines how OntoSynth built-in role agents, skills, and review pages work inside OpenClaw Desktop.

Design goals:

1. OpenClaw stays the runtime control plane (`agents`, `workers`, `tools`, `cron`, `workshop`).
2. OntoSynth contributes built-in course-creation roles and domain workflow.
3. Every phase produces a user-reviewable artifact before promotion to the next phase.

## Canonical Mapping

1. OntoSynth skill -> capability bundle assigned to a role agent.
2. OntoSynth role agent -> OpenClaw agent identity (persistent).
3. OntoSynth phase run -> OpenClaw worker/session execution.
4. OntoSynth pipeline task -> OpenClaw workshop task (manual) or cron job (scheduled).

## Built-in Skills and Role Agents

1. `course-structure` -> `os.course-structure-agent`
2. `course-knowledge` -> `os.course-knowledge-agent`
3. `course-mindmap` -> `os.course-mindmap-agent`
4. `course-assessment` -> `os.course-assessment-agent`
5. `course-slide` -> `os.course-slide-agent`
6. Optional control skill `course-orchestration` -> `os.orchestrator-agent`

## Phase Lifecycle

Each phase follows the same lifecycle:

1. `draft`: generated by agent run.
2. `edited`: user modified content.
3. `approved`: user accepted artifact for handoff.
4. `rejected`: user rejected artifact and requested regeneration.

Gate rule:

1. Next phase can only start from latest `approved` artifact of previous phase.

## Artifact Envelope

Every phase artifact uses this shared envelope:

```json
{
  "artifactId": "string",
  "courseId": "string",
  "phase": "structure|knowledge|mindmap|assessment|slides",
  "version": 1,
  "status": "draft|edited|approved|rejected",
  "createdAt": "ISO-8601",
  "updatedAt": "ISO-8601",
  "createdByAgentId": "os.*",
  "approvedBy": "user-id-or-device-id",
  "sourceRefs": [],
  "content": {},
  "meta": {}
}
```

## Per-Skill Interaction Pages

Each skill page in `Course Studio` should provide:

1. Input panel (phase config + constraints)
2. Run panel (run/re-run with selected role agent)
3. Output preview panel
4. Edit panel (form + raw JSON/markdown mode)
5. Diff/history panel (compare versions)
6. Approve/Reject controls

Page order:

1. Structure
2. Knowledge
3. Mindmap
4. Assessment
5. Slides

## Gateway/Bridge RPC Contract

Minimum methods:

1. `ontosynth.agents.syncBuiltins`
2. `ontosynth.agents.listBuiltins`
3. `ontosynth.pipeline.create`
4. `ontosynth.pipeline.runPhase`
5. `ontosynth.artifacts.list`
6. `ontosynth.artifacts.get`
7. `ontosynth.artifacts.update`
8. `ontosynth.artifacts.approve`
9. `ontosynth.artifacts.reject`
10. `ontosynth.artifacts.history`

## Execution Rules

1. Use role agents only (never create one agent per course/application).
2. Tag all synced OntoSynth agents with `managed:ontosynth`.
3. Preserve user model overrides unless `forceModel=true`.
4. Treat artifact approval as explicit handoff boundary.
5. Keep all phase outputs versioned and reversible.

## Rollout Plan

1. Phase 1: sync built-in agents and expose `agents.list` in Workshop selector.
2. Phase 2: add `Course Studio` shell with read-only artifact previews.
3. Phase 3: add editing + approve/reject actions.
4. Phase 4: enforce gate rules and cron templates for scheduled rebuilds.
